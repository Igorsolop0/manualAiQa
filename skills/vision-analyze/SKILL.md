# Робота з зображеннями

Агент може "бачити" зображення завдяки мультимодальним моделям.
Цей скіл налаштовано на використання `zai/glm-4.5v`, оскільки ця модель офіційно підтримує обробку зображень через API і є високопродуктивною для аналізу скріншотів та картинок (DeepSeek-VL поки що не має офіційного API в загальному доступі, а GLM-4.5V вже вбудована у Z.AI).

**Як працює процес:**
1. Користувач завантажує зображення (PNG/JPG) у Slack або Telegram.
2. OpenClaw (через плагіни) зберігає файл та генерує локальний шлях.
3. Коли ти бачиш у повідомленні шлях до зображення і прохання його проаналізувати (або описати), ти маєш використати інструмент `vision.analyze`.
4. В аргумент `image_path` передай абсолютний шлях до файлу (який був збережений плагіном).
5. Дочекайся відповіді від візуальної моделі (`glm-4.5v`) з детальним описом або відповіддю на супутнє запитання.

**Рекомендації:**
- Якщо користувач хоче розпізнати текст на зображенні, опиши йому знайдений текст словов-в-слово.
- Якщо користувач скидає скріншот з багом, проаналізуй UI-елементи та опиши, що пішло не так.
