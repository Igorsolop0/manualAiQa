## Objective
Verify that frontend correctly passes auth token during Telegram account linking flow.

## Execution Policy
**Stagehand REQUIRED** - Use Stagehand for browser discovery and interaction. Do not fallback to Playwright generation.

## Acceptance Criteria
- ✅ Frontend sends auth token when initiating linking
- ✅ Backend correctly receives token and includes in OAuth
- ✅ Linking flow works end-to-end for Telegram
- ✅ Error handling implemented (missing token, invalid token)

## Test Scope

### Priority 1 - Critical Path
1. **TC-1: Auth token in request (Happy Path)**
   - Verify `linkToken` present in `/TelegramAccount/HashAuth` request
   - Token matches current session
   - Response 200 OK

2. **TC-2: Successful Telegram linking (E2E Flow)**
   - Complete OAuth flow
   - Verify Telegram account shows as "Linked"
   - Verify callback with token in Network

3. **TC-4: Error Handling - Missing Token**
   - Clear auth token from storage
   - Trigger linking
   - Verify error message and redirect to login

### Priority 2 - Secondary
4. **TC-3: Unlink Telegram account**
   - Verify unlink flow works
   - Button changes to "Link Telegram"

5. **TC-5: Error Handling - Invalid Token**
   - Modify token to invalid value
   - Verify 401 error and user-friendly message

6. **TC-8: Security - Token Not Exposed in URL**
   - Verify token NOT in URL query params
   - Token only in POST body or headers
